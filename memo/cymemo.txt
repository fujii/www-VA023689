記憶を頼りに私と Emacs について記す。

emacs にはまる第1ステップは、BS, RET, 矢印キーを
使わなくなったことだ。
これはいつ頃だろうか？多分2年のときだろう。
3年になって IRIX を使えるようになり私が真っ先に
したのが Caps を Ctrl にすることだったから。
このころは Ctrl + (1文字) の組合せ闇雲に押して勉強した。
C-t が条件付けられたのもこの時代である。
C-t よく思いついたもんんだ。別になくてもいい C-t。

この時代について詳しく述べる。
体はすでに emacs しか受け付けなくなっていた。
BS の代わりの C-h などはほぼ無意識に押した。
家の PC では win が linux にリプレイスされた。
しかし、私は emacs にはまっていなかった。
それは、「emacsから出ない」が理解できなかったのだ。
emacsユーザはemacsから出ようとしない。すべてを
emacs内で完結させようとする。
この話は知っていたが、私はすべてを emacs でしたいとは
思っていなかった。
なぜなら、もしもそうしてしまうと頻繁にバッファを
切替えなくてはならないからだ。
そう、私はバッファの切替が嫌いだった。
そのためそのころに使っていた MUA は sylpheed だった。
EBView や netscape を愛用し、マウスでさばいていた。
この時代が長かった。

転機が訪れたのは、4年のときだった。
4年になって navi2ch を知り、それまでやや避けていた
2ch.net にはまってしまった。
その日もいつの通り日課となった2chのチェックを
していたことだろう。そこで出会ったのが、
「自慢の.emacsを貼り付けよう」[*1]だった。
そこで以下に出会った。

  ;;C-x bでbuffersを選ぶ時便利
  (iswitchb-default-keybindings)

それまでのバッファ切替えの悩みに対する解決だった。

今では上の様にして、これを頻繁に使っている。
現在愛用の elisp は wl の他に lookup, emacs-w3m, migemo,
navi2ch である。

背骨はS字に曲がっているが、この椅子の前の方に座ると
腰のところが曲がり過ぎてしまう。

人もいたが、その人はどうなったのだろう。
痛みの期間は1週間目から1カ月目だっただろうか。

Distribution: global

elisp の info の mapcar のページで見つけた便利な関数。

(defun mapcar* (function &rest args)
  "Apply FUNCTION to successive cars of all ARGS.
          Return the list of results."
  ;; リストをつくしていなければ
  (if (not (memq 'nil args))
      ;; CARに関数を適用する
      (cons (apply function (mapcar 'car args))
	    (apply 'mapcar* function
		   ;; Recurse for rest of elements.
		   (mapcar 'cdr args)))))

(mapcar* 'cons '(a b c) '(1 2 3 4))
  => ((a . 1) (b . 2) (c . 3))

 Timingbuffered disk reads:  64 MB in 28.13 seconds =  2.28 MB/sec

 /dev/hda:
  setting using_dma to 1 (on)
   using_dma    =  1 (on)

   /dev/hda:
    Timing buffered disk reads:  64 MB in  1.99 seconds = 32.16 MB/sec

その差、14倍。

KTerm*eightBitInput: false

Date: 2002-04-11
Subject: kterm: Alt キーを有効にしたい
Distribution: global

KTerm*VT100*Translations:   #override \
          Alt<KeyPress>: string(0x1b) insert-seven-bit()

これを .Xresouces などに記入すると Alt+X で "ESC X" が
入力される。
