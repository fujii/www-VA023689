Subject: kinput2 と FreeWnn で辞書を追加
Distribution: global

私家版pubdic+補遺[*1]をインストールして使用している。
しかし、kinput2 & を .xsession に追加してから
どうも egg で supplement が有効にならなかった。

これはどうやら、kinput2 で supplement を使用していない
ことが問題でした。そこで kinput2 でも supplement を
使用する方法。

~/.Xresources に下の1行追加して、 xrdb .Xresources

  Kinput2*CcWnn.wnnEnvrc4: /home/fujii/etc/wnn/wnnenvrc

次に

$ cp /usr/share/wnn/ja_JP/wnnenvrc ~/etc/wnn/wnnenvrc

してそのファイルに下の1行追加。

  setdic misc/supplement.dic     usr/@USR/supplement.h     1 1 0 - - 0

これで良さそう。

[*1] FreeWnn/Wnn4.2/Eggを賢くするものたち
  http://www.remus.dti.ne.jp/~endo-h/wnn/

Subject: emacs: emacs21で日本語が印刷できない
Distribution: global

(setq ps-multibyte-buffer 'non-latin-printer)

として、C-u M-x ps-print-buffer をすると
変な postscript が出力される。

http://tsukuba.m17n.org/mule-ja-archive/2002-3/msg00008.html

にあるように、

(require 'ps-mule)
(defalias 'ps-mule-header-string-charsets 'ignore)

も追加すると、本文はでてくる。

元に戻して、

http://tsukuba.m17n.org/mule-ja-archive/2001-11/msg00011.html

より、postscript をエディタでいじって、

  /h0 14.000000 /Ryumin-Light-H DefAsciiFontMule 

を

  /h0 14.000000 /Ryumin-Light-EUC-H DefAsciiFontMule 

にすると、本文の日本語はうまくいく。また、

http://lists.debian.or.jp/debian-users/200111/msg00248.html

の方法で、コメントアウトしてもうまくいく。

  %/h0 14.000000 /Ryumin-Light-H DefAsciiFontMule

Date: 2002-06-28
Subject: unix: $HOME に etc を
Distribution: global

ホームディレクトリの下に etc と var が欲しいとよく思う。
そして、$HOME に現在63個もあるドットファイルとドットディレクトリ(?)を
そこに移動したい。

$ alias cd='HOME=$HOME cd'
$ export HOME=$HOME/etc

とすれば、うまくいくのではとふと思ったが恐くてできない。

~/{etc,var} を作る利点を考えてみる

 - rm .* の被害者これ以上出さずにすむかも
 - alias ls='ls -a' としやすい
 - バックアップが楽になるかも

問題点
 
 - ./configure --prefix=$HOME としている人は etc が衝突する

ちなみに既に ~ の下に bin と usr と tmp はある。opt はない。
よって普段は、 ./configure --prefix=$HOME/usr って感じでやってる。
