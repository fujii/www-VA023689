LaTeX に取り込みたい。

これはあまりスマートでない。そこで /usr/share/texmf を
ひたすらして grep して見つけた。

まず、プリアンブルに以下を追加。

  \usepackage{verbatim}

そして挿入したいところにこんな感じ。

  \verbatiminput{filename.txt}
  \verbatiminput*{filename.txt}

どうも、TAB が無視されているみたい。

行の終りで改行がされない。

Webを検索すると \verbatimlisting とか、
\verbatimfiles なんてのもあるみたい。

記憶を頼りに私と Emacs について記す。

演習で mule を使うから、それを使い始めた。

使わなくなったことだ。
これはいつ頃だろうか？多分2年のときだろう。
3年になって IRIX を使えるようになり私が真っ先に
したのが Caps を Ctrl にすることだったから。
このころは Ctrl + (1文字) の組合せ闇雲に押して勉強した。
C-t が条件付けられたのもこの時代である。
C-t よく思いついたもんんだ。別になくてもいい C-t。

この時代について詳しく述べる。
体はすでに emacs しか受け付けなくなっていた。
BS の代わりの C-h などはほぼ無意識に押した。
家の PC では win が linux にリプレイスされた。
しかし、私は emacs にはまっていなかった。
それは、「emacsから出ない」が理解できなかったのだ。
emacsユーザはemacsから出ようとしない。すべてを
emacs内で完結させようとする。
この話は知っていたが、私はすべてを emacs でしたいとは
思っていなかった。
なぜなら、もしもそうしてしまうと頻繁にバッファを
切替えなくてはならないからだ。
そう、私はバッファの切替が嫌いだった。
そのためそのころに使っていた MUA は sylpheed だった。
EBView や netscape を愛用し、マウスでさばいていた。
この時代が長かった。

4年になって navi2ch を知り、それまでやや避けていた
2ch.net にはまってしまった。
その日もいつの通り日課となった2chのチェックを
していたことだろう。そこで出会ったのが、
「自慢の.emacsを貼り付けよう」[*1]だった。
そこで以下に出会った。

  ;;C-x bでbuffersを選ぶ時便利
  (iswitchb-default-keybindings)

それまでのバッファ切替えの悩みに対する解決だった。
その後 electric-buffer-list とも出会った。
水を得た魚では例えがおかしいが、その後は

MUA が wl に代わるのは時間の問題だった。

emacs にはまったのだからと elisp も勉強を始めた。
ふと計算したいときは *scratch* を関数電卓として使いだした。

  ;; electric-bufffer-list
  (define-key global-map "\C-x\C-b" 'electric-buffer-list)
  (add-hook 'electric-buffer-menu-mode-hook 'forward-line)

今では上の様にして、これを頻繁に使っている。
現在愛用の elisp は wl の他に lookup, emacs-w3m, migemo,
navi2ch である。

を参照。これを読んで欲しくなりました。

よって、いつから使い始めたのか正確には覚えていないのだが、
3月の8日くらいだったと思う。

記憶が曖昧になってはきているが、まずはその身体的苦痛について。
まず痛くなったのが臑(すね, 脛)である。
ここが痛くなることは事前の調査と予測により分かっていたし、
すぐに慣れるだろうと思っていた。実際そうだっし、
痛みのほども大したことはなかった気がする。
痛みの期間は2日目から2週間くらい。

次に悩まされたのは腰の痛みである。これは予想外であった。
たぶん腰が痛くなった原因は、腰の不自然な反り過ぎである。
背骨はS字に曲がっているが、この椅子の前の方に座ると
腰のところが曲がり過ぎてしまう。
これは結構痛かった。腰痛の人でこの椅子を考慮している
人もいたが、その人はどうなったのだろう。
痛みの期間は1週間目から1カ月目だっただろうか。

そして、これまた予想外なところに痛みがでた。
それは多分僧帽筋と呼ばれる筋肉である。
背中の真中より上で、肩甲骨のすぐ下あたりだったと思う。
なぜここが痛くなったのかは分からない。
本を読むために下を向いているときによく痛くなった気がする。
そうすると、机が低く過ぎで、下を向き過ぎていたのだろうか。
その頃よく腕立てなどの筋トレをしていたのが原因かもしれない。
そろそろ、筋トレを再開してみて確認してみようか。
この痛みの期間は2週間目から1カ月半目までだったかな。

2.5カ月の現在ではもう痛みに襲われることはなくなった。
しかしながら、満足しているわけでもない。
この椅子は普通の椅子よりも落ち着かない気がする。
じっとしていられないのはいつもの事だが、
この椅子ではさらにじっとしていられない、そんな気がする。
じっとしていられないのは、不自然な姿勢をして体に負担が
かかっている事を体が私に教えてくれているのかも知れない。

とはいえ高かった椅子だし、やっとまともに座れるようになったし、
この椅子はなれるのに1年くらいかかっても不思議ではないので
まだまだ使用を続ける。

Subject: idea: マウスのホイールを押したときの挙動
Distribution: global

Windows においてマウスのホイールを押すと、上下の矢印のような
丸いなにかが表示されて、マウスの動きとスクロールが連動される。
この連動の仕方を変更すると良い気がする。
現在は押した場所、つまり丸いなにかが表示された場所からの
距離がスクロールの速度に比例している。
私の提案ではこの丸いなにかからの距離とスクロール量を比例させる。
つまり、マウスを上下するとそれに併せて画面は下へ上へとスクロールする。
ただ単にスクロールバーをドラッグすればいいのだが、
ボタンの押しっぱなしが疲れる。それに左右にもスクロールできるし。

別の案としては、モーダレスにしてホイールを押している間、
スクロールとマウスの移動を連動する。

探せば、こんなツールはありそうだな。

Subject: emacs: latex: コンパイルのメッセージのウィンドウの高さを指定
Distribution: global

(setq compilation-read-command nil)

上のような設定で、M-x compile を C-c c に割り当てることにより、
コンパイルのウィンドウの高さの指定と毎回のコマンドの入力を無しにしている。

しかし、latex をコンパイルする C-c C-f (tex-file) は
ウィンドウの高さを指定できなかったので以下を .emacs.elに追加。

(eval-after-load "tex-mode"
'(defun tex-display-shell ()
  "Make the TeX shell buffer visible in a window."
  (require 'compile)
  (compilation-set-window-height
   (display-buffer (tex-shell-buf)))
  (tex-recenter-output-buffer nil))
)

ちょっと使ってみた感想。
M-x compile は 15 行でいいのだが、platex はメッセージが多いので
15行では狭いかも。とりあえずは、これで使用してみる。

debian にはソースがインストールされていないようだ。
emacs21-elisp-sources なんてパッケージが欲しいかも。

まずは tcp の接続を listen するようにする。
/etc/X11/xdm/Xservers の

:0 local /usr/X11R6/bin/X vt7 -dpi 100 -nolisten tcp

を

#:0 local /usr/X11R6/bin/X vt7 -dpi 100 -nolisten tcp

とする。そして、Ctrl + Alt + BS にて X の再起動。
local のマシンで、

local$ xauth list

で出てきた行からそれらしい1行を選んで、マウスで選択。
remote のマシンで

remote$ xauth add local.foo.bar.baz:0  MIT-MAGIC-COOKIE-1  7aa9ab0d195ecac397cb4838d900aec9

でクッキーを登録

remote$ xclock

で xclock が local の X ホストに出てくるのを確認できたら、OK。

Subject: book: 本はどう読むか

Date: 2002-04-21
Subject: emacs: mapcar*
Distribution: global

elisp の info の mapcar のページで見つけた便利な関数。

(defun mapcar* (function &rest args)
  "Apply FUNCTION to successive cars of all ARGS.
          Return the list of results."
  ;; リストをつくしていなければ
  (if (not (memq 'nil args))
      ;; CARに関数を適用する
      (cons (apply function (mapcar 'car args))
	    (apply 'mapcar* function
		   ;; Recurse for rest of elements.
		   (mapcar 'cdr args)))))

(mapcar* 'cons '(a b c) '(1 2 3 4))
  => ((a . 1) (b . 2) (c . 3))

Subject: guile: コマンドラインで readline を有効に
Distribution: global

guile-1.4では以下を ~/.guile に記述。
(use-modules (ice-9 readline))
(activate-readline)

guile-1.5 では[*1]
This doesn't work in guile-1.5.x out of the box because the default curent module for the repl is
now (guile-user). The workaround is to place in your .guile file:
(define-module (guile-user))

     "Why does readline break going from 1.4 to 1.5.x?"
